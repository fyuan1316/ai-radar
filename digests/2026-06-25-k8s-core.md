# K8s 自身核心周报 2026-06-25

窗口:2026-06-18 → 2026-06-25

## 摘要(3 条以内)

1. **k8s v1.37.0-alpha.2 落地两条对产品有直接影响的默认行为变更**:HPA 默认支持 scale-to/from-zero、Relaxed DNS for Services GA;同时 kube-proxy 开始为"默认切 nftables"铺路(iptables 用户将收到 warning)。详见 https://github.com/kubernetes/kubernetes/releases/tag/v1.37.0-alpha.2
2. **运行时层一次大规模安全升级**:containerd 同日修 5 个 CVE 并全分支(1.7.33 / 2.0.10 / 2.1.9 / 2.2.5 / 2.3.2)放补丁;runc 发首个稳定版 v1.5.0 并收缩老分支支持(1.2.z 即刻 EOL,1.3.z 仅高危 CVE)。基础镜像/节点需排期升级。https://github.com/containerd/containerd/releases/tag/v2.3.2
3. **KEP 侧最值钱的几条**:in-place node resource resize(KEP-3953,跨多 SIG 推进)、DRA preemption 新提案(KEP-5690)、CSI Sidecars All-in-one(KEP-4958)、HPA fallback strategy(KEP-5679)。这些决定未来 6 个月的资源效率与调度架构选型。

## K8s 核心版本节奏

- **v1.37.0-alpha.2 发布(2026-06-25)**,1.37 周期推进中。本期 alpha.2 关键能力变更:
  - **HPA scale to/from zero 默认开启**(SIG Autoscaling),缩到 0 再拉起成为开箱能力——对无状态推理/批服务的成本优化是利好。https://github.com/kubernetes/kubernetes/pull/139648
  - **Relaxed DNS names for Services 升 GA**(SIG Network)。https://github.com/kubernetes/kubernetes/pull/139282
  - **WatchListCompression Beta 默认开**:WatchList 响应对带 `Accept-Encoding: gzip` 的 client 做 gzip 压缩,降 apiserver 出口带宽。https://github.com/kubernetes/kubernetes/pull/139308
  - **WebhookRoundTripLoadBalancing Beta 默认开**:`--enable-aggregator-routing=true` 时 admission webhook 请求在所有 endpoint 间均衡,避免连接缓存把并发全打到单后端。https://github.com/kubernetes/kubernetes/pull/139237
  - **CBOR 作为 discovery / 结构化错误编码**(`CBORServingAndStorage` 门控)。https://github.com/kubernetes/kubernetes/pull/139632
  - **kube-proxy 默认模式迁移信号**:kubeadm 现显式把空 `mode` 落为 `iptables`,1.37 起未显式设置将告警,为未来默认切 `nftables` 铺路。升级前要确认我们产品默认值。https://github.com/kubernetes/kubernetes/pull/139777

## 调度(kube-scheduler)

- **PVC 挂载调度性能优化**:对带 PersistentVolumeClaim 的 Pod,调度周期间只处理 delta 计数,降大规模有状态负载的调度开销。https://github.com/kubernetes/kubernetes/pull/139238
- Bug 修复:清空 NominatedNodeName 时 pod 被错误挂在空 node key 下(nominator 状态泄漏);assumed pod 删除时正确从 podGroup 缓存移除。https://github.com/kubernetes/kubernetes/pull/139904
- KEP 动向(详见末节):TopologySpread 排除全 taint 域(KEP-5980)、PreScore/Rescoring 流程澄清(KEP-5598)。
- 注:GangScheduling / WorkloadAwarePreemption 门控被移除、合并为 GenericWorkload —— 属 workload-aware/gang 范畴,归 `k8s-ai-infra`,此处仅备注。

## 节点 & kubelet

- **kubelet subPath 挂载在 FUSE/GlusterFS 网络存储中断后卡死错误循环** 已修。https://github.com/kubernetes/kubernetes/pull/139275
- **node shutdown manager 反复失败泄漏 dbus 连接** 导致线程耗尽崩溃,已修。https://github.com/kubernetes/kubernetes/pull/137141
- ephemeral-storage capacity 改用 DecimalSI,修 capacity 与 allocatable 格式不一致。https://github.com/kubernetes/kubernetes/pull/137652
- cri-api:KeyValue value 字段 JSON 编码回退到 1.34 前行为(避免兼容性破坏)。https://github.com/kubernetes/kubernetes/pull/139964

## Runtime(containerd / CRI-O / runc)

- **containerd 安全补丁波(2026-06-18,全分支同日)**:v1.7.33 / v2.0.10 / v2.1.9 / v2.2.5 / v2.3.2,修 5 个 CVE(CVE-2026-50195 / 53488 / 53492 / 53489 / 47262);另含 Windows shim 日志读取 data race、镜像分发 last-host 重试、并发 task RPC 超时致容器启动失败等修复。**建议尽快排期升级节点运行时**。https://github.com/containerd/containerd/releases/tag/v2.3.2
