### VPA

- **[vertical-pod-autoscaler-1.7.0](https://github.com/kubernetes/autoscaler/releases/tag/vertical-pod-autoscaler-1.7.0)(2026-05-29)正式发布**,核心变更:

## API Machinery & Operator 生态

本周 API Machinery 是非 AI 侧的"主战场":

- **[KEP-5366 ControllerManagerReleaseLeaderElectionLockOnExit → beta](https://github.com/kubernetes/enhancements/pull/6142)**(2026-06-02 合并) —— 控制器优雅退出时主动 release leader lock。**意义**:所有用 client-go leader election 的组件(包括 controller-runtime 写的 operator)滚动升级 / 重启时,切主延迟从"等 lease 超时(默认 15s+)"降到"立即"。**这是 1.37 直接可享的运维提速**。
- **[KEP-5975 Declarative API Definitions](https://github.com/kubernetes/enhancements/pull/5974)**(2026-06-01 合并,XL size,jpbetz 主导) —— 给内置 API 与 CRD 引入声明式定义机制(原 schema-as-data 思路的演进),长期目标是把 Go 代码侧的 API 定义与 OpenAPI / CRD schema 合一。**对所有写 operator 的人都是结构性影响**,值得读 KEP。
- **[KEP-5958 Client Opt-out for managedFields in API Response](https://github.com/kubernetes/enhancements/pull/6015)**(2026-06-01 合并) —— 客户端可在 List/Get 时选择不返回 managedFields,显著降流量与解析开销(对 controller list 大量对象的场景尤其重要)。
- **[KEP-5793 manifest-based admission config → beta](https://github.com/kubernetes/enhancements/pull/6093)** 持续推进。
- **[KEP-5966 EtcdRangeStream](https://github.com/kubernetes/enhancements/pull/5967)**(本周更新,联挂多 SIG) —— 大规模 list/watch 的 etcd 流式接口。
- controller-runtime / kubebuilder / operator-sdk **无新 release**;controller-runtime 主分支本周只合一个 dependabot([#3527](https://github.com/kubernetes-sigs/controller-runtime/pull/3527));kubebuilder 合 [#5674 CLI 支持 k8s 1.36](https://github.com/kubernetes-sigs/kubebuilder/pull/5674) + [#5708 helm/v2-alpha 默认 NetworkPolicy 模板](https://github.com/kubernetes-sigs/kubebuilder/pull/5708);operator-sdk 无窗口活动。
