## 摘要

- [Kubernetes v1.36.0](https://github.com/kubernetes/kubernetes/releases/tag/v1.36.0) 于 2026-04-22 发布,这是本周核心底座事件。启示:我们需要立刻更新产品支持矩阵,明确 v1.36 对 scheduler、kubelet、admission、CRD、DRA、Gateway API 相关依赖的影响。
- [Karpenter v1.12.0](https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.12.0) 于 2026-04-24 发布,新增 do-not-disrupt annotation grace period,并修复静态 provisioning、node label re-sync、already-bound pods timestamp 等问题。启示:节点自动扩缩容进入“更少打断 workloads”的精细化阶段,AI 训练/推理平台要把 disruption 控制策略暴露给用户。
- [Ray 2.55.1](https://github.com/ray-project/ray/releases/tag/ray-2.55.1) 修复 `ray-llm` image SSH connectivity,而 [Ray 2.55.0](https://github.com/ray-project/ray/releases/tag/ray-2.55.0) 刚加入 Ray Data vLLM metrics export 和 Data LLM Grafana dashboard。启示:Ray 正在把 LLM 数据处理/推理链路的可观测性做进官方栈,我们集成 Ray 时要关注镜像安全、指标和 dashboard,不仅是 CRD 安装。

## K8s 核心版本节奏

- [Kubernetes v1.36.0](https://github.com/kubernetes/kubernetes/releases/tag/v1.36.0) 已发布,release 页面指向 CHANGELOG 和 kubernetes-announce。
  - 启示:这是一个需要产品层同步的 minor release:测试矩阵、operator SDK/controller-runtime 版本、CRD conversion webhook、集群升级文档都应进入本周排期。
- 上周同时出现 [v1.35.4](https://github.com/kubernetes/kubernetes/releases/tag/v1.35.4)、[v1.34.7](https://github.com/kubernetes/kubernetes/releases/tag/v1.34.7)、[v1.33.11](https://github.com/kubernetes/kubernetes/releases/tag/v1.33.11) patch release。
  - 启示:客户环境会同时停留在 1.33-1.36 多个 minor。我们的支持矩阵不能只写“支持 Kubernetes”,要精确到 minor + patch + known issues。

## 调度(kube-scheduler)

- 本次未能通过 API 扫描 kube-scheduler 和 sig-scheduling KEP 的本周 PR。已确认的调度相关高信号来自 [Karpenter v1.12.0](https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.12.0) 的 do-not-disrupt grace period。
  - 启示:对 AI workloads,调度器只是第一层,节点生命周期管理的 disruption policy 同样影响训练/推理 SLA。我们应该把“不可打断窗口/宽限期/自动 checkpoint”作为统一产品概念。

## 节点 & kubelet

- 本次未能完成 kubelet PR 扫描。Kubernetes v1.36.0 需要补读 [CHANGELOG](https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG/CHANGELOG-1.36.md) 中 sig-node 项。
  - 启示:AI 节点通常绑定 GPU driver、device plugin、runtime class、hugepage/NUMA 配置。升级 v1.36 前必须做节点层 e2e,不能只跑控制面 smoke test。

## 自动扩缩容(VPA / CA / Karpenter)

- [Karpenter v1.12.0](https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.12.0) 新增 do-not-disrupt annotation grace period,并修复 static provisioning predicate race、registration hooks 后 node label re-sync、already-bound pods timestamp 等问题。
  - 启示:对 GPU 节点池,grace period 能降低误中断训练/推理副本的风险。我们应该支持把用户侧 SLA 转成 Karpenter disruption 配置,并在控制台展示“此 workload 是否允许缩容打断”。
- Cluster Autoscaler / VPA 本次未能通过 API 扫描到高置信本周信号。
  - 启示:VPA 仍需与 in-place resize、Kueue、GPU request 组合评估。下次补扫 `kubernetes/autoscaler`。

## 网络 & 存储(大盘)

- [Gateway API monthly-2026.04](https://github.com/kubernetes-sigs/gateway-api/releases/tag/monthly-2026.04) 是实验通道月度 release,包含 Gateway API main 分支最新特性和修复。
  - 启示:OAI 3.x 已经把 dashboard URL 迁到 Gateway 路径,我们自己的平台如果也要统一入口、模型网关、MCP/Playground API,应尽早把 Gateway API 放进北南向流量架构。
- [Cilium v1.19.3](https://github.com/cilium/cilium/releases/tag/v1.19.3) 是 2026-04-15 的 patch release,修复 L7 policy proxy redirect、KVStore identity mode、BGP service advertisement race、policy memory leak、NodePort DSR 等问题。
  - 启示:Cilium patch 对企业 AI 集群很重要:长连接推理流量、Gateway/L7 policy、Hubble 可观测性都会被 CNI bug 影响。平台支持矩阵应明确 Cilium patch 级别。
