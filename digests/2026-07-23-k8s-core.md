# K8s 自身核心周报 2026-07-23

窗口:2026-07-16 → 2026-07-23

## 摘要(3 条以内)

1. **v1.37 进入 beta**:`v1.37.0-beta.0` 于 07-20 切出(code freeze 临近),一批 feature gate 晋级落定——ClusterTrustBundle 系列 **GA**、`metrics.k8s.io` v1beta1→**v1 GA**、HPAConfigurableTolerance **GA**、PVCUnusedSinceTime **beta**;in-tree 的 WorkloadAwarePreemption / PodGroup 调度栈继续成型(新增 `PodGroupPostFilter` 扩展点)。https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG/CHANGELOG-1.37.md
2. **原地扩容(in-place resize)向调度器纵深**:新增 alpha gate `InPlacePodVerticalScalingSchedulerPreemption`,scheduler 会为高优先级 pod 的 `Deferred` 原地扩容抢占低优先级 pod——这是 VPA 真正生产可用(KEP-1287 GA 之后)的关键补齐。https://github.com/kubernetes/kubernetes/pull/140000
3. **网络控制面开始长新资源**:两份 alpha KEP 提案——`LoadBalancer` 资源(KEP-6128,显式管理/监控 LB)与 `EndpointSelector` 资源(KEP-6116,无需 Service 直接选 pod 端点);Cilium `v1.20.0-rc.1` 把 MCS-API(多集群 Service)提到 **stable**。

## 调度(kube-scheduler)

本周调度改动几乎全在 **in-tree Workload-Aware Preemption / PodGroup** 这条线(与 Kueue/gang 有交集,归属见文末边界说明,这里只记对核心调度框架的结构性影响):

- **新扩展点 `PodGroupPostFilter`**:把原先硬编码的 WorkloadAwarePreemption 逻辑抽成可配置扩展点;PodGroup 整组不可调度时才跑,常规 PostFilter 不再对 PodGroup 内 pod 触发。https://github.com/kubernetes/kubernetes/pull/139674 https://github.com/kubernetes/kubernetes/pull/140412
- **API 面**:`scheduling.k8s.io/v1alpha3` 新增 `CompositePodGroup` https://github.com/kubernetes/kubernetes/pull/139596 ;`PodGroupTemplate` 加 `PreemptionPolicy` 字段 https://github.com/kubernetes/kubernetes/pull/140312 。
- **抢占行为**:Pod-by-Pod 抢占支持把 PodGroup 作为 victim https://github.com/kubernetes/kubernetes/pull/137981 ;WorkloadAwarePreemption 改为"移除所有潜在 victim 后只做一次调度尝试"以换性能(牺牲最优 victim 选择)https://github.com/kubernetes/kubernetes/pull/139980 。
- **指标**:`scheduler_plugin_execution_duration_seconds`、`scheduler_scheduling_algorithm_duration_seconds` 从 alpha → **beta**;新增 `queued_entities` / `queue_incoming_entities_total`(按 pod/podgroup 统计队列)。https://github.com/kubernetes/kubernetes/pull/138176
- KEP 侧:KEP-5598「opportunistic batching + rescoring」两 PR 合并 https://github.com/kubernetes/enhancements/pull/6039 ;KEP-5075 更新 1.37 beta 实现历史 https://github.com/kubernetes/enhancements/pull/6254 。

## 节点 & kubelet

- **cgroup 修复**:burstable pod 的 `memory.low`(软保护)此前因父 cgroup 缺祖先层覆盖而失效,已修——影响所有用 memory soft protection 的节点。https://github.com/kubernetes/kubernetes/pull/140267

## 自动扩缩容(VPA / CA / Karpenter)

- **Cluster Autoscaler**:**DRA 支持在 1.35 线成型**——补齐 CA DRA 支持文档 https://github.com/kubernetes/autoscaler/pull/10010 、新增 DRA autoscaling E2E https://github.com/kubernetes/autoscaler/pull/10036 ;CapacityBuffer 特性修 replicas/百分比 sizing 逻辑 https://github.com/kubernetes/autoscaler/pull/9959 并补 helm 权限 https://github.com/kubernetes/autoscaler/pull/9933 ;clusterapi provider 实现 scale-up 时的 machine phase 校验 https://github.com/kubernetes/autoscaler/pull/10001 。CA 版本推进:1.36.1 / 1.35.2 / 1.34.5 / 1.33.6。
- **Karpenter**:`v1.14.0` 上周(07-11)已发;本周实质 PR:**DRA prioritized alternatives 支持** https://github.com/kubernetes-sigs/karpenter/pull/3142 、修 capacity/overhead override 语义 https://github.com/kubernetes-sigs/karpenter/pull/3151 。CA 与 Karpenter 双双补 DRA,是本周自动扩缩容的共同方向。
