# K8s 自身核心周报 2026-06-18

窗口:2026-06-11 → 2026-06-18

## 摘要(3 条以内)

1. **v1.37 周期正式开闸**:`v1.37.0-alpha.1` 已出(2026-06-11),四条维护线 v1.33.13 / v1.34.9 / v1.35.6 / v1.36.2 同日打补丁。1.37 的两条主线已能从 alpha.1 + KEP 看清:一是 **kube-scheduler 把"工作负载感知调度 / PodGroup"做进核心**(队列、抢占、PlacementScore 全部 PodGroup 化),二是 **apiserver watch 缓存大改**(EtcdRangeStream beta、并发解码 beta、PostStartHook GA)。
2. **网络栈方向确定**:kube-proxy 的 ipvs 模式被正式标记弃用(自 1.35),新增 `KubeProxyIPVS` 特征门为后续摘除铺路;nftables 后端(1.33 GA)成为推荐路径,KEP-5343「nftables 设为默认后端」本周推进。升级到新内核的集群需要开始规划 ipvs→nftables 迁移。
3. **runc 安全补丁**:CVE-2026-41579(低危,`/dev` 符号链接可对宿主机文件系统做受限写)无禁运公开披露,v1.4.3 / v1.3.6 / v1.5.0-rc.3 三线同发,建议节点 runtime 跟随升级。 https://github.com/opencontainers/runc/security/advisories/GHSA-xjvp-4fhw-gc47

## 调度(kube-scheduler)

本周最大主题是 **"工作负载感知调度 / PodGroup" 正在从 scheduler-plugins 外置插件下沉进 in-tree kube-scheduler**——这是 1.37 调度器的中心叙事:

- **调度队列 PodGroup 化**:active/backoff/unschedulable 三队列抽象为 `QueuedEntityInfo`,可装单 Pod 或 PodGroup。https://github.com/kubernetes/kubernetes/pull/138567
- **PodGroup 抢占成型**:成功抢占后给 PodGroup 内 Pod 设 NominatedNodeName(https://github.com/kubernetes/kubernetes/pull/138967);抢占时尽量让更多 preemptor Pod 可落位(https://github.com/kubernetes/kubernetes/pull/138757);victim 已在 nominated 节点终止时跳过冗余抢占(https://github.com/kubernetes/kubernetes/pull/138710)。
- **新扩展点**:`PlacementFeasible`(GangScheduling 插件用它在 minCount 不可满足时提前终止 PodGroup 调度周期,https://github.com/kubernetes/kubernetes/pull/138643);`PlacementCycleState` + `PlacementScore`,挂在 alpha `TopologyAwareWorkloadScheduling` 门下(https://github.com/kubernetes/kubernetes/pull/138274)。
- **框架细节**:`PatchPodStatus` 改收 `[]*v1.PodCondition` 切片,避免并发条件互相覆盖(https://github.com/kubernetes/kubernetes/pull/135160);Binding API 调用遇瞬时错误会重试(https://github.com/kubernetes/kubernetes/pull/138855);新增三种 Pod 事件子类型 `AssignedPod`/`UnscheduledPod`/`TargetPod`,插件可精确注册以提升性能(https://github.com/kubernetes/kubernetes/pull/135905)。

对应 KEP 群(本周合入):Workload-aware preemption beta 1.37(KEP-5710,https://github.com/kubernetes/enhancements/pull/6106)、Topology-aware workload scheduling beta 1.37(KEP-5732,https://github.com/kubernetes/enhancements/pull/6134)、Workload Aware Scheduling Controller APIs(KEP-6089,https://github.com/kubernetes/enhancements/pull/6092)、CompositePodGroup API(KEP-6012,https://github.com/kubernetes/enhancements/pull/6017)。

> 边界说明:gang/PodGroup 与 `k8s-ai-infra`(scheduler-plugins gang、Kueue)主题重叠,但这一批是 **in-tree kube-scheduler 核心机制改造**,会影响所有用调度器的工作负载,故在本任务记录核心机制面;AI 专用编排仍归 `k8s-ai-infra`。

## 节点 & kubelet

- **MemoryQoS → Beta(1.37)**:cgroup v2 内存 QoS 长期卡 alpha,本周毕业 beta,生产可用度上一个台阶。https://github.com/kubernetes/enhancements/pull/6143
- **Node Lifecycle Conditions(KEP-5683)进 alpha**:为节点引入标准化生命周期条件,配套 Specialized Lifecycle Management。https://github.com/kubernetes/enhancements/pull/6161
- **kubelet 弃用时间线右移**:配置 flag 及 fallback 行为的移除从 1.37 推迟到 1.38,以对齐 containerd v1.7 支持窗口。https://github.com/kubernetes/kubernetes/pull/139121
- **EvictionRequest API 更新**(KEP-4563,https://github.com/kubernetes/enhancements/pull/6074);**Default Pod Sysctls in Kubelet**(KEP-5996);**WritableCgroups option**(KEP-5474);**Topology Manager 单 NUMA 节点密度优化**(KEP-6007,https://github.com/kubernetes/enhancements/pull/6008)。
- 新增安全 sysctl 白名单项 `net.ipv4.tcp_slow_start_after_idle`、`tcp_notsent_lowat`(https://github.com/kubernetes/kubernetes/pull/138389);kubelet 卷指标 `storage_operation_duration_seconds` 等升 Beta(https://github.com/kubernetes/kubernetes/pull/136189)。
- kubeadm:`ipvs` 模式延迟弃用告警上线,提示新内核用户迁 nftables(https://github.com/kubernetes/kubernetes/pull/139067);`NodeLocalCRISocket` 特征门 GA 后移除(https://github.com/kubernetes/kubernetes/pull/138645)。

## 自动扩缩容(VPA / CA / Karpenter)

- 推荐质量修复:不再丢弃聚合边界附近的新鲜 OOM 信号,避免 OOM 后推荐不上调。https://github.com/kubernetes/autoscaler/pull/9725

**Cluster Autoscaler**:

- 修 "Skipping scale down on node group" 日志刷屏(https://github.com/kubernetes/autoscaler/pull/9804);Azure VMSS 删除失败后的尺寸缓存处理(https://github.com/kubernetes/autoscaler/pull/9779);GCE 新实例错误处理改进(https://github.com/kubernetes/autoscaler/pull/9826)。

**Karpenter**:本周三个 PR,信号明确——**Karpenter 开始做 DRA**:

- **声明式校验**:`DeclarativeValidationTakeover` 门锁定不可再设(https://github.com/kubernetes/kubernetes/pull/139212);新增 `+k8s:dependentRequired` 声明式校验 tag(https://github.com/kubernetes/kubernetes/pull/139164);CEL 准入评估改零分配反射,降 CPU/GC(https://github.com/kubernetes/kubernetes/pull/138771)。新 KEP-6164「内部版本消除」立项(https://github.com/kubernetes/enhancements/pull/6165)。
- **授权**:authorizer 可返回条件性决策(Allow/Deny/NoOpinion 之外,https://github.com/kubernetes/kubernetes/pull/137204);新增 `UnconditionalAuthorizer` 窄接口(https://github.com/kubernetes/kubernetes/pull/138801)。新 KEP-6060「apiserver 到 webhook 的认证」立项(https://github.com/kubernetes/enhancements/pull/6156)。
- **kubebuilder v4.15.0**:支持 K8s 1.36 / Go 1.26,controller-runtime 升 v0.24.1;Helm 插件特征开关命名 `enable`→`enabled`(破坏性,需 `kubebuilder edit --plugins=helm/v2-alpha --force`);生成项目默认带 NetworkPolicy、pin ENVTEST 版本。https://github.com/kubernetes-sigs/kubebuilder/releases/tag/v4.15.0

## 值得跟进

- [ ] **读 Workload-aware scheduling KEP 群**(KEP-5710 / 5732 / 6089 / 6012):in-tree kube-scheduler 的 PodGroup/拓扑感知一旦 beta,我们产品里基于 scheduler-plugins gang 的方案需要评估迁移路径与并存策略。
- [ ] **盯 in-place resize 抢占线**(KEP-5836 1.37 alpha + KEP-3953):这是 VPA 生产化的最后拼图,关系到我们资源效率叙事——评估在自家产品中何时打开。
- [ ] **规划 kube-proxy ipvs→nftables 迁移**:1.37 起 `KubeProxyIPVS` 门 + 弃用告警已就位,梳理我们交付集群里仍用 ipvs 的存量,制定迁移窗口。
- [ ] **apiserver watch 缓存升级影响评估**(EtcdRangeStream beta + 并发解码默认开):大规模集群 apiserver 启动/一致性读会变,纳入 1.37 升级回归测试。
- [ ] **runc 升级**:节点镜像里 runc 升到 v1.4.3+(或对应 1.3.6),关掉 CVE-2026-41579。
- [ ] **升级前清理 `scheduling.k8s.io/v1alpha2` 对象**:1.37 整组换 v1alpha3,写进升级 runbook。
