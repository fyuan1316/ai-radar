## 摘要

- [Kubernetes v1.36.0](https://github.com/kubernetes/kubernetes/releases/tag/v1.36.0) 在 2026-04-22 发布,这是本周最大的底座事件。启示:所有 AI 侧 DRA/Kueue/LWS/JobSet 方案都要重新核对与 v1.36 的兼容矩阵,尤其是 CRD conversion、scheduler、admission 和 runtime 依赖。
- [KServe v0.18.0-rc1](https://github.com/kserve/kserve/releases/tag/v0.18.0-rc1) 在 2026-04-22 发布,LLMInferenceService 继续补齐 LWS、WVA/KEDA/HPA、llm-d 0.6、ModelCache namespace scope、GatewayClass 测试等能力。启示:KServe 正在把 LLM 专用运行时能力收进 release train,我们需要跟进 v0.18 GA。
- [Kueue v0.17.1](https://github.com/kubernetes-sigs/kueue/releases/tag/v0.17.1) 虽在 2026-04-16 发布,但对 AI 集群仍是本周重点:MultiKueue、TAS、LWS integration、ElasticJobs、AdmissionChecks 都有修复。启示:Kueue 已经进入“多集群 + 拓扑感知 + LWS/训练作业”稳定性阶段,应作为默认批调度底座评估。

## GPU / 加速器管理

- [NVIDIA GPU Operator v26.3.1](https://github.com/NVIDIA/gpu-operator/releases/tag/v26.3.1) 在 2026-04-18 发布,接近本周窗口,官方 release 页面指向 GPU Operator 26.3.1 release notes。
  - 启示:GPU Operator 的企业部署节奏仍然快于很多平台集成节奏。我们需要把 GPU Operator 版本、驱动版本、CUDA runtime、K8s minor 版本作为安装前兼容性检查,不要只依赖 Helm chart 安装成功。
- [Node Feature Discovery 最新 release v0.18.3](https://github.com/kubernetes-sigs/node-feature-discovery/releases/tag/v0.18.3) 当前公开 release 仍停留在 2025-11,本周未见新 release。
  - 启示:NFD 本周不是变化源,但它仍是 GPU/NPU/NUMA/CPU feature 发现的通用层。我们的异构资源抽象要避免把硬件 label 写死在各个 operator 里。

## 调度 & 资源分配

### Kueue

- [Kueue v0.17.1](https://github.com/kubernetes-sigs/kueue/releases/tag/v0.17.1) 修复 AdmissionChecks、MultiKueue、ElasticJobs、FailureRecoveryPolicy、TAS、LWS integration 等问题。
  - 启示:这不是新功能周,而是生产化修复周。尤其 MultiKueue 的远端 workload eviction/fallback 修复,说明多集群训练/批任务调度已经进入真实故障场景,我们若做跨集群训练调度,应直接复用并测试 MultiKueue 的 failure semantics。
- [Kueue v0.17.1](https://github.com/kubernetes-sigs/kueue/releases/tag/v0.17.1) 的 LWS integration 修复 PodTemplate metadata 未传播到 Workload PodSets。
  - 启示:LWS + Kueue 联动已经不是概念验证,而是开始修细节。对多节点推理,我们应该把 LWS/Kueue 组合当成默认路径,而不是自己造 gang controller。

### DRA

- [Kubernetes DRA 文档](https://kubernetes.io/docs/concepts/scheduling-eviction/dynamic-resource-allocation/) 仍是 GPU/加速器未来接口的核心参考。本次未能通过 API 扫出本周新增 DRA KEP/PR。
  - 启示:DRA 相关信息本周需要补扫。产品上不要把 device-plugin 扩展写死成唯一方向,资源池化、设备分片、claim 参数化都应预留 DRA 接入点。

### LWS / JobSet

- [KServe v0.18.0-rc1](https://github.com/kserve/kserve/releases/tag/v0.18.0-rc1) 在 llmisvc 中包含“resolve LWS leader address for ZMQ”“handle missing LWS CRD gracefully”“autoscaling for WVA and KEDA/HPA integration”等变更。
  - 启示:KServe 正在把 LWS 作为 LLMInferenceService 的多节点执行模型。我们做 KServe 对接时要验证 LWS CRD 不存在、CRD 版本不匹配、leader 地址变化这些边界条件。
- [LWS v0.8.0](https://github.com/kubernetes-sigs/lws/releases/tag/v0.8.0) 虽不是本周发布,但其 release notes 已包含 VolumeClaimTemplates、PersistentVolumeClaimRetentionPolicy、Kueue user guide、Kubeflow Trainer integration、MaxUnavailable beta 等。
  - 启示:LWS 从“多副本编排”向“有状态/训练/调度集成”演进。我们应把 LWS 看成 AI workload primitive,而不是只作为 llm-d 依赖。
- [JobSet v0.11.1](https://github.com/kubernetes-sigs/jobset/releases/tag/v0.11.1) 本周无新 release,最新 patch 只修 PVC 等稳定性问题。
  - 启示:JobSet 本周不是热点;训练编排当前更值得跟 Kubeflow Trainer v2 + Kueue + LWS 组合。
