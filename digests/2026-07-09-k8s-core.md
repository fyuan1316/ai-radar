# K8s 自身核心周报 2026-07-09

窗口:2026-07-02 → 2026-07-09

## 摘要(3 条以内)

1. **v1.37.0-alpha.3 落地(07-08),一批 GA 晋级已进主线**:`SELinuxMount` GA 且默认开启(SELinux 集群升级前必须先自查工作负载,否则可能挂载失败)、`StorageVersionMigration` GA(StorageMigration/v1)、node declared features GA、`kubectl -o kyaml` stable;kubelet 内嵌 cAdvisor 换成精简 lib 模块,**一批 `--*` 旧 flag 和 `container_cpu_load_*` 等指标被移除**,升级前要清理 kubelet 配置和监控看板。https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG/CHANGELOG-1.37.md
2. **Cluster Autoscaler 1.36.0 是本周最有分量的自扩缩容更新**:新增 `CapacityQuota` CRD(给 CAS 上量化配额上限)、实验性 `--salvo-scale-up`(单个 loop 内多次扩容)、DRA partitionable devices 支持与一组 DRA 指标、fastpath binpacking。对我们做资源治理/多租户配额有直接参考价值。https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.36.0
3. **kube-proxy 开始为 iptables→nftables 默认切换预热**:未显式指定 proxy mode 时会告警,未来某版本 Linux 默认从 iptables 切到 nftables——这是会影响所有集群网络平面的节奏点,需提前纳入升级验证。https://github.com/kubernetes/kubernetes/pull/139957

## K8s 核心版本节奏

**v1.37.0-alpha.3 发布,2026-07-08**(pre-release)。1.37 周期已合入的关键 API 变更 / 晋级:

- **`SELinuxMount` → GA**,1.37 默认开启。SELinux 集群升级前请按官方博客自查会被打断的工作负载,可选择修复或临时 opt-out;非 SELinux 集群无影响。https://github.com/kubernetes/kubernetes/pull/139956
- **`StorageVersionMigration` → GA**,配套 StorageMigration/v1 API,默认开启;SVM condition 增加 progress 展示已迁移对象数。对 CRD/etcd 存储版本平滑迁移有用。https://github.com/kubernetes/kubernetes/pull/138560
- **node declared features → GA**。https://github.com/kubernetes/kubernetes/pull/139763
- **Conditional Authorization 核心机制落地(alpha 实现)**,对应 sig-auth 的 KEP-6001 结构化授权决策方向。https://github.com/kubernetes/kubernetes/pull/137513
- **WatchList 响应压缩 → Beta**(`WatchListCompression` gate,默认开),客户端带 `Accept-Encoding: gzip` 时 apiserver 返回 gzip;大规模 informer 场景可减带宽。https://github.com/kubernetes/kubernetes/pull/140140
- `kubectl get -o kyaml` 晋级 stable。https://github.com/kubernetes/kubernetes/pull/140076
- 修复 SSA(server-side apply)在 patch container 类型列表/map 时可能误报 `422 required` 的回归。https://github.com/kubernetes/kubernetes/pull/140294

## 调度(kube-scheduler)

- 本周主线 alpha.3 里调度侧改动**大量集中在 in-tree PodGroup / Workload-Aware Scheduling**(PodGroup API、`PreemptionPolicy`、`WorkloadRef`、调度队列 `incompletePodGroupPods`、requeue 到 active queue 等一连串 PR)。**这条线归 `k8s-ai-infra`(gang scheduling / WVA),本任务不展开**——仅提示:PodGroup 已作为核心 API 类型进入 1.37 主线,后续会影响所有用 PodGroup 的调度插件生态。https://github.com/kubernetes/enhancements/pull/6092
- 通用调度器修复(与 AI 无关,值得知道):修复"Pod 已成功调度却卡在 `PodScheduled=False`"的 bug(https://github.com/kubernetes/kubernetes/pull/139602);修复 PodGroup 调度周期内 inter-pod (anti-)affinity 与 volume restriction 评估、AssumePod/ForgetPod 快照维护(https://github.com/kubernetes/kubernetes/pull/139054)。
- 默认抢占(preemption)找到候选节点时,`FailedScheduling` 事件与 `PodScheduled` condition 现在会带 "found a potential placement … preempting N victims" 提示,排障更直观。https://github.com/kubernetes/kubernetes/pull/140180

## 节点 & kubelet

- **cAdvisor 精简化(DEPRECATION,影响升级)**:kubelet 内嵌 cAdvisor 改用更精简的 `cadvisor/lib` 模块,移除一批长期弃用面:
  - 一堆旧 flag 不再接受,**设置了 kubelet 会启动失败**(仅保留 `--housekeeping-interval`),如 `--containerd`、`--container-hints`、`--storage-driver-*` 等,升级前须从 kubelet 配置里删掉;
  - `/stats/summary` 的 `userDefinedMetrics`、`/metrics/cadvisor` 的 `container_application_*` 自定义指标不再采集;
  - `container_cpu_load_average_10s`、`container_cpu_load_d_average_10s`、`container_tasks_state` 三个指标不再导出——**监控看板/告警要提前改**。https://github.com/kubernetes/kubernetes/pull/139870
- **`HugepageAwareEviction`(默认开启)**:有 hugepages 的节点上,eviction manager 计算 `memory.available` 时正确排除 hugepage 预留 RAM,修复因可用内存虚高导致的延迟驱逐 / OOM。可用 gate 关闭回退。https://github.com/kubernetes/kubernetes/pull/138127
- 修复 1.36 引入的 **kubelet 内存泄漏回归**(每次 Pod sync 泄漏 context)——在 1.36 上的用户值得关注这个补丁。https://github.com/kubernetes/kubernetes/pull/139850

## 自动扩缩容(VPA / CA / Karpenter)

- **Cluster Autoscaler 1.36.0(2026-07-08,minor)**,重点能力:
  - **`CapacityQuota` CRD**:限制 CAS 扩容出的资源总量,`--capacity-quotas-enabled` 开启,后台带 reconcile loop——多租户/成本护栏的原生手段。https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.36.0
  - 实验性 **`--salvo-scale-up`**:单个 CA loop 内触发多次扩容,`--salvo-scale-up-budget` 控时间预算,提升大规模突发扩容速度。
  - **DRA:partitionable devices 支持** + 新增 DRA ScaleUp/ScaleDown 指标、`dra_node_template_resources_mismatch` 指标、修复 DRA 多 resourceslice 节点就绪判断。
  - fastpath binpacking 扩容优化;新增 `--max-startup-time`、`--startup-taint-prefix`;status configmap 新增 `suspended` 状态。
  - 同期还发了补丁版 1.34.4 / 1.33.5 / 1.35.1。
- **VPA**:本周无新 release(最新仍是 1.7.0 / 2026-05-29)。**继续盯 in-place pod resize(KEP-1287)线**——GA 后 VPA 才在生产真正可用,本周主线无该 KEP 状态变更。
- **Karpenter**:本周把**全部支持分支同步打了补丁**(v1.11.2、v1.10.1 … 一直到 v1.0.9,均 2026-07-08),内容是 bug fix(Node Repair 场景防止 pod grace period 变负值),无新特性。https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.11.2
- **Cluster API**:本周无 release。
