# K8s 自身核心周报 2026-07-16

窗口:2026-07-09 → 2026-07-16

## 摘要(3 条以内)

1. **v1.37 周期主线是"workload-aware scheduling / PodGroup"进核心**:kube-scheduler 原生 PodGroup API 大量落 alpha(新增 PreemptionPolicy 字段、PodGroupTemplateRef→WorkloadRef 重构、一致性快照/队列),这是过去只在 scheduler-plugins(gang)里做的能力第一次进 upstream 主调度器,对我们所有批量/AI 工作负载调度都是长期基座。见 https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG/CHANGELOG-1.37.md
2. **v1.37 一批破坏性 GA 要提前排升级**:`SELinuxMount` 晋 GA 且默认开(SELinux 集群可能打断现有工作负载),`StorageVersionMigration` GA、node declared features GA、`kubectl -o kyaml` stable;kube-proxy 已开始预警默认模式将从 iptables 切 nftables。https://github.com/kubernetes/kubernetes/pull/139956
3. **Karpenter v1.14.0 把 DRA 打通到节点供给层**:新增 DRA allocator + consumable capacity/partitionable devices 支持,以及 CapacityBuffer v1beta1 API 和 Balanced 整合策略——Karpenter 现在能按 ResourceClaim 需求算该起什么机型,是我们做 GPU 弹性节点池的关键一环。https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.14.0

## K8s 核心版本节奏

- 当前开发主线 **v1.37**,最新切的是 **v1.37.0-alpha.3**(2026-07-08,恰在窗口边缘)。稳定线 v1.36.2、v1.35.6 为最新 patch。
- 本周期已确认的 **GA / 破坏性变更**(进 v1.37):
  - **SELinuxMount 晋 GA、默认开启**——SELinux 启用的集群升级前必须按官方博客排查会被 volume 打标影响的工作负载,否则可能挂载失败。https://github.com/kubernetes/kubernetes/pull/139956
  - **StorageVersionMigration GA**(StorageMigration/v1),默认开启,SVM 条件现带迁移进度。https://github.com/kubernetes/kubernetes/pull/138560
  - **node declared features 晋 GA**。https://github.com/kubernetes/kubernetes/pull/139763
  - **DRA Device Taints & Tolerations GA**(resource.k8s.io/v1)——归 k8s-ai-infra,但作为 DRA 核心协议对所有 ResourceClaim 用户有溢出影响,在此备一笔。https://github.com/kubernetes/kubernetes/pull/138676
  - **kubectl get -o kyaml** 提升为 stable。https://github.com/kubernetes/kubernetes/pull/140076
- **Conditional Authorization** 核心机制首次落地(luxas),为后续条件化授权铺路。https://github.com/kubernetes/kubernetes/pull/137513

## 调度(kube-scheduler)

本周期调度侧几乎全部围绕 **workload-aware scheduling / 原生 PodGroup**(把 gang 语义抬进 upstream 主调度器):

- **API 变更**:PodGroup 新增 `PreemptionPolicy` 字段(定义 workload 级抢占策略);`PodGroupTemplateRef` 重构为更直接的 `WorkloadRef`(对齐未来 CompositePodGroup);`MinCount` 变为可后改;条件 `PodGroupScheduled`→`PodGroupInitiallyScheduled` 语义澄清。https://github.com/kubernetes/kubernetes/pull/139240
- **调度器内部**:PodGroupInfo/SharedLister 暴露一致 PodGroup 状态、新增 `incompletePodGroupPods` 队列结构、成组调度后剩余 pod 直接回 active queue 保时间戳优先级、优先级一致性校验。https://github.com/kubernetes/kubernetes/pull/140077
- **多处成组调度 bug 修复**:共享 ResourceClaim 的 PodGroup 卡调度/被排到无 claim 节点、成组周期内 inter-pod affinity 与 volume restriction 评估错误、抢占事件消息增强("found a potential placement…")。https://github.com/kubernetes/kubernetes/pull/139054
- **KEP 更新**(enhancements 仓,窗口内):KEP-5710 加 `PodGroupPreemptionPolicy` feature gate(https://github.com/kubernetes/enhancements/pull/6222)、KEP-6089 / KEP-5547 按实现回填。两条关闭的 KEP 值得留意方向——KEP-5721 给 Tolerations/NodeAffinity 加语义化版本比较运算符、KEP-5500 给 Tolerations 加 CEL 表达式支持(https://github.com/kubernetes/enhancements/pull/5822)。

> 边界说明:gang scheduling 在 scheduler-plugins 里的部分归 k8s-ai-infra;但 **PodGroup 作为 upstream 主调度器的通用原语**影响所有批量负载用户,故在本任务记录。

## 节点 & kubelet

- **eviction 修复(默认开)**:`HugepageAwareEviction`(默认开)让 kubelet 的 `memory.available` 正确排除 hugepage 预留 RAM,修复被"虚高可用内存"拖慢的驱逐/OOM。https://github.com/kubernetes/kubernetes/pull/138127
- **内存泄漏回归修复**:修 1.36 引入的 kubelet 每次 Pod sync 泄漏 context 的内存泄漏。https://github.com/kubernetes/kubernetes/pull/139850
- **sig-node KEP 动向**(enhancements 窗口内):
  - KEP-6063:每 Pod PID 下限从 1024 降到 128(小集群/高密度更友好)。https://github.com/kubernetes/enhancements/pull/6217
  - KEP-4929:给 lifecycle hook 的 ExecAction 加超时。https://github.com/kubernetes/enhancements/pull/6235
  - KEP-6245:驱逐时用 surge 容量替代"被 PDB 阻塞",提升驱逐流畅度。https://github.com/kubernetes/enhancements/pull/6245
  - KEP-5683 Specialized Lifecycle Management、KEP-6232 容忍 per-NUMA 内存漂移、KEP-6122 DownwardAPIAssignedResources 停用行为更新。

## 自动扩缩容(VPA / CA / Karpenter)

- **Karpenter v1.14.0**(2026-07-11,重磅):
  - **DRA 全链路**:dra allocator 实现与集成、consumable capacity + partitionable devices 支持、按 ResourceClaim 需求决定机型(`HasDRARequirements` 识别 pod 级 claim),并修了 ResourceClaim 水合死锁与 DRA 计数竞态。
  - **CapacityBuffer API 升 v1beta1**、发 `operator_*` 状态指标——预留缓冲容量做主动扩容。
  - **Balanced 整合策略**、`consolidateAfter` 现作用于整合的目标节点、daemonset 调度纳入机型需求。https://github.com/kubernetes-sigs/karpenter/releases/tag/v1.14.0
- **VPA**(autoscaler mono-repo,窗口内 merge):给 in-place update 的 deferred 日志加原因(vpa-release-1.7 也 backport)、为 `resourcePolicy.containerPolicies.controlledResources` 加校验、修 Recreate 模式下内存请求随用量增长的 flake。**in-place pod resize(KEP-1287)本窗口无 GA 级进展**,VPA 仍在围绕 in-place 更新补日志/校验,继续观察。https://github.com/kubernetes/autoscaler/pull/9945
- **Cluster Autoscaler**:RunOnce 循环下传 context、修 scale-up 模拟中 `equivalence.PodGroup` 对跳过节点组的误变异、GCE MIG 缓存/URL 解析性能优化。https://github.com/kubernetes/autoscaler/pull/9827
- **Cluster API v1.14.0-beta.0**(2026-07-15):bump 到 **controller-runtime v0.24 + controller-tools v0.21 + Go 1.26**;拆分独立 API module(减少下游对 kube-openapi 的直接依赖);新增实验命令 `clusterctl convert`;managedFields interning、stale controller 缓解。4 处破坏性变更(CAPD docker 资源标记 deprecated 等)。https://github.com/kubernetes-sigs/cluster-api/releases/tag/v1.14.0-beta.0
