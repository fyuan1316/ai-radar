## Runtime(containerd / CRI-O / runc)
- [containerd v2.3.0](https://github.com/containerd/containerd/releases/tag/v2.3.0) — **首个年度 LTS,支持 ≥2 年**,LTS 之间(1.7→2.3)直升测试支持
  - Highlights: container fs copy 的 transfer types(#13165)、log 注入 trace ID(#13117)、OTel trace 在 plugin 出向 RPC 传播(#13113)、Sandbox API 加 spec 字段(#12840)
  - CRI: 容器在 host network 下使用 user namespace(#12518)、UpdatePodSandboxResources 接 Sandbox API(#13118)、按运行时 per-layer label 解包(#12835)
  - **EROFS 路线在加速**:zstd-wrapped EROFS 层(#13185)、native 镜像 os.features 支持(#13091)、EROFS layer media type(#12567)、EROFS snapshotter 加 dmverity(#12502)
  - 启示:containerd 走 4 个月节奏 + 年度 LTS,这是企业 K8s 升级节奏的"地基锚点";EROFS 在 containerd 端持续投入,长期是 squashed-image / 大镜像快速分发的方向;AI 镜像普遍很大(数 GB 训练镜像),要持续盯
- cri-o 本窗口无新版(v1.35.2 在 2026-04-02)
- runc 本窗口无新版(v1.5.0-rc.2 / v1.4.2 在 2026-04-03)
